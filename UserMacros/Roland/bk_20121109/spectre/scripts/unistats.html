%unistats

Introduction

There are two major reporting macros in Spectre - %npcttab and %unistats. These two macros are intended to create the bulk of safety and demography tables. The macros have some claims to being called "standard macros" for safety reporting across the pharmaceutical industry because they are free to download and use, will run on any platform and have enough features built into them to satisfy the reporting needs of most of the industry. This page is about %unistats. Its name gives a clue as to what it does; the "uni" comes from "proc univariate" and the "stats" are the statistics it calculates. These are "descriptive statistics" such as "N", "Min", "Mean", "Median", "Max" and "Standard Deviation". It also calculates category counts and percentages, but this is not represented in its name. This macro does not produce a table by default. This is because the output data can be presented in many different ways. There is a common way of producing the output, however, but you must ask the macro to give you this output. Features of the macro are listed later.

To see the design specification for %unistats, click here.

%unistats calls other macros. For a list of dependencies, click here.

Note: If you see a misalignment of the columns in any example of the output shown below then this is caused by the html editor I am using and is not caused by a bug in the reporting macro.

Automation using %unistats

The macro is aimed at making automatic reporting possible so it is designed to be extremely easy to call. By setting only three parameters, dsin=, varlist= and unicatrep=yes it will produce a report. Though the macro has 90+ parameters that can be set, you are expected to use the minimum. Most of the parameters are there to fine tune the layout and most of these you will never use. Because of the large number of parameters, learning how to use the macro from reading the header is difficult, just as it is for the %npcttab macro. You will need a simple introduction to the macro and that is what you will find on this page. Once you have learned to use the macro in a simple way then you will be able to benefit from reading the macro header. Once you feel you would like to look at the macro in more detail, you can link to it below as well as the major sub-macros it calls to calculate p-values and produce a report.
%unistats
%unipvals
%unicatrep

%popfmt and %unistats

The macro %popfmt has to be called before %unistats. These two macro are connected. When you calculate percentages for categorical variables, this percentage will be based on the treatment population by default, rather than overall counts. For this it will use the _popfmt dataset created by the %popfmt macro call. To calculate percentages based on overall category counts instead, you will have to use the setting pctcalc=cat.

If you use the %unicatrep macro to display output, either directly using unicatrep=yes or after a call to %unistats, then %popfmt will have told the macro what variable names to expect. It also informs %unicatrep about the column widths to use by default. You have to know the %popfmt macro well to get the best out of %unistats. You can link to a page about it here.

Features of %unistats

  • Calculates summary statistics (for numeric variables) and category counts with percentages for categorical variables.
  • Can optionally print a report if unicatrep=yes is set.
  • Can produce a correctly formatted report with only three parameters set
  • Automatically displays the total for all treatment arms if total=yes is set.
  • Will by default fit the page width if a print is requested.
  • Will plug missing values with zeroes.
  • Percentages for categories will be based on the treatment arm population by default. Percentages based on total category counts can be requested using pctcalc=cat.
  • Indentation can be adjusted.
  • Spacing between treatment arms can be adjusted.
  • Will produce output datasets that are already transposed for convenience.
  • Allows the number of decimal places to be set for different numeric variables.
  • Aligns decimal points for descriptive statistics unless dpalign=no specified
  • Automatically maps free-style statistics labels to their statistics keyword.
  • Will display categories in format order if a format is applied.
  • Will turn the second character onwards to lower case if requested for variables defined to lowcasevarlist= parameter.
  • Page flow can be controlled.
  • Will display all format values with missing ones having a zero count for individual or all variables.
  • Will calculate p-values for Fisher's Exact test and Chi-square test for categorical variables, F-test and Student's t test for numeric variables and Kruskal-Wallis test or the Wilcoxon rank sum test for non-parametric numeric variables.
  • Writes p-values to global macro variables so that they can be used in footnotes instead of displayed in a column.

Demonstrating %unistats

You will be introduced to the features of %unistats using demonstrations showing you code and output. You must have already set up two datasets in your sasuser area as described on this page. If you have not done this then please exit this page and do so.

There are several demonstrations and each of them will build on the previous demonstrations. It would be a very good idea if you tried this out yourself using an interactive SAS® software session. The demonstrations will not use the macros %openrep or %closerep so you will see characters that look like a "p" but with the bulge slipped down. Do not worry about these characters. They are there to help column alignment and they will get converted to spaces when the %closerep macro is called. In fact it is useful to see these characters are there because sometimes you will want to use this character yourself.

If you are using a laptop for these demonstrations or your PC at home then the following code will function using SAS® Learning Edition. You will learn more if you actually submit the code and see the results and SAS Learning Edition is an inexpensive and worthwhile alternative to owning a full copy of SAS software.

Demo 1

The first demonstration is a very basic one. You will see a very simple call to the macro and the output it produces. Please try all these demonstrations yourself by submitting the code in an interactive SAS session. You will have to make sure that sasautos is locating the macros.
 
/* Demonstrate %unistats: program 1 */

options noovp nodate nonumber center 
missing=" " formchar='|_---|+|---+=|-/\<>*';

options sasautos=("C:\spectre\macros" SASAUTOS);

options ls=78 ps=63;

title1;

proc format;
  value racecd
  1="CAUCASIAN"
  2="BLACK"
  3="ASIAN"
  4="HISPANIC"
  5="OTHER"
  ;
  value sexcd
  1="MALE"
  2="FEMALE"
  ;
  value trtcd
  1="Ambicor (1g/day)"
  2="Betamaxin (500mg/day)"
  3="No treatment"
  ;
  value trtnarr
  1="Ambicor@(1g/day)"
  2="Betamaxin@(500mg/day)"
  3="No@treatment"
  ;
  value NY
  0="NO"
  1="YES"
  ;
run;
 

proc sort data=sasuser.demog(where=(fascd=1))
           out=demog(drop=fascd);
  by patno invid;
run;
 

%popfmt(demog,trtcd,uniqueid=patno invid)

%unistats(dsin=demog,unicatrep=yes,
stats=N Min Mean Max STD.,
varlist=sexcd racecd age weight height);

Here is the output produced by this very simple call. It could be better. The labels are all in capital letters and the top label is truncated. The macro uses the variable labels and since these were set to capital letters then it uses them. The gender code and race code are the coded values and these values are in capitals because the format it uses maps these codes to capital letter descriptions. Looking at the output you can see one clever feature of the macro. It has used information from the call to %popfmt to set the column widths so the "Betamaxin" column fits well.

Demo 6

Looking at all the outputs so far, displaying the minimum and maximum age with a decimal point is not a good idea as age is an integer number of years. The format for MIN and MAX can be changed to an integer format by setting minfmt=3. and maxfmt=3. . This has an effect on all the numeric values, though, so we will need to adjust this for weight and height. Here is the first attempt at this. We will see that it is not ideal. Look in the macro call for the "/+1" added onto the end of the weight and height variables.

Demo 12

Here is the change to the code to display the "0.131" value in the first race variable and to delete the second race variable. Also, the footnote symbol needs to be changed and a new footnote added.

We are now in the realm of calling %unistats without it producing a report and manipulating the output dataset and then calling the %unicatrep macro to finish the report for us. It is starting to get complicated! We need to know about the structure of the output datasets. I will tell you at this point that each variable has an order and that this gets assigned to the variable _varord and each statistic for that variables has an order defined to_statord.

For this demo I am going to do something extra. Since we are playing with the output file, I will show you how to combine the "min" and "max" values for height into a "range" and display it where the "min" value would have been displayed. If I were doing this for all the variables then all I would have to do is to combine the stats labels like "Min;Max" and the macro would do the work. However, in this case it will only be applied to the height variable so that method can not be used. Decimal point alignment will be lost in most cases but it is useful in showing you how to manipulate the output dataset to combine values if you need to. This is a simple case where I know the variable lengths are big enough to incorporate these two values combined.
 
/* Demonstrate %unistats: program 12 */

options noovp nodate nonumber center 
missing=" " formchar='|_---|+|---+=|-/\<>*';

options sasautos=("C:\spectre\macros" SASAUTOS);

options ls=78 ps=63;

title1;

proc format;
  value racecd
  1="CAUCASIAN"
  2="BLACK"
  3="ASIAN"
  4="HISPANIC"
  5="OTHER"
  ;
  value sexcd
  1="MALE"
  2="FEMALE"
  ;
  value trtcd
  1="Ambicor (1g/day)"
  2="Betamaxin (500mg/day)"
  3="No treatment"
  ;
  value trtnarr
  1="Ambicor@(1g/day)"
  2="Betamaxin@(500mg/day)"
  3="No@treatment"
  ;
  value NY
  0="NO"
  1="YES"
  ;
  value agecat
    0-15="<16 yrs"
   16-25=" 16 - 25 yrs"
   26-40=" 26 - 40 yrs"
   41-65=" 41 - 65 yrs"
 66-high=">65 years"
  ;
run;
 

proc sort data=sasuser.demog(where=(fascd=1))
           out=demog(drop=fascd);
  by patno invid;
run;

data demog;
  set demog;
  agecat=age;
  racecd2=racecd;
  if racecd2 ne 1 then racecd2=5;
  format trtcd trtnarr. agecat agecat. racecd2 racecd.;
  label sexcd="Gender"
        racecd="Race"
        racecd2="Race"
        agecat="Age (yrs)"
        age="A0"x
        weight="Weight (kg)"
        height="Height (cm)"
        ;
run;

%popfmt(demog,trtcd,uniqueid=patno invid)

%lafootnote(1,"%sysfunc(repeat(_,200))")
%lafootnote(2,"~ Chi-square test")
%lafootnote(3,"° Fisher's Exact test comparing Caucasian with all other categories combined")
%lafootnote(4,"^ Fisher's Exact test")
%lafootnote(5,"# Student's t test")

%unistats(dsin=demog,total=yes,
lowcasevarlist=sexcd racecd racecd2,
stats=N Min Mean Max STD.,
minfmt=3.,maxfmt=3.,
varlist=sexcd racecd racecd2 agecat age weight/m height/m,
pvarlist=sexcd racecd racecd2 agecat age weight height,
chisqvarlist=sexcd,
fishervarlist=racecd racecd2 agecat,
allcatvars=racecd agecat,
dstrantrt=tran)

*- this step applies the p-value -;
data tran;
  set tran;
  if _varord=2 and _statord=1 then delete;
  else if _varord=3 then do;
    if _statord ne 1 then delete;
    else do;
     _varord=2;
     substr(TRT9999,7,1)="°";
    end;
  end;
run;

*- this step creates the height range -;
data tran;
  length h1 h2 h99 $ 11;
  retain h1 h2 h99 " ";
  set tran;
  if _varord=7 and _statord=2 then do;
    h1=trt1;
    h2=trt2;
    h99=trt99;
    delete;
  end;
  if _varord=7 and _statord=4 then do;
    _statord=2;
    _statlabel="Range";
    trt1=trim(scan(h1,1," "))||"-"||scan(trt1,1," ");
    trt2=trim(scan(h2,1," "))||"-"||scan(trt2,1," ");
    trt99=trim(scan(h99,1," "))||"-"||scan(trt99,1," ");
  end;
run;

%unicatrep(dsin=tran,indent=3,
trtlabel="Number of Patients (%) /" "Descriptive Statistics" " ");

Demo 13

You can control the way p-values are displayed. It will display them in a column by default but it is easy to override this. The p-values are written to global macro variables named _pvalue1_ , _pvalue2_ etc. where the number is the same as the _varord value. Suppose I only wanted to show a few p-values in the footnotes then I could do it like this.
